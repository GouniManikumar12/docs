# Getting Started as an AI Platform

Welcome to AdMesh! This guide will help you integrate intelligent product recommendations into your AI agent, chatbot, or AI-powered platform.

---

## What is AdMesh?

AdMesh is an **AI-native advertising protocol** that provides contextually relevant product recommendations for AI agents. When users ask questions, AdMesh returns products that match their intent, and you earn revenue when users engage.

**Example**: User asks "What's the best CRM for startups?" → AdMesh returns relevant CRM products → User clicks/converts → You earn commission

## Make Your First API Call

### Basic Request

```bash cURL
curl -X POST "https://api.useadmesh.com/agent/recommend" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "best CRM for startups",
    "format": "auto"
  }'
```

### Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | ✅ | User's search query or question |
| `format` | string | ❌ | Response format (`auto`, `product`, `sidebar`, `conversation`) |
| `previous_query` | string | ❌ | Previous query for conversation context |
| `previous_summary` | string | ❌ | Summary of previous conversation |
| `session_id` | string | ❌ | Session identifier for tracking |
| `is_fallback_allowed` | boolean | ❌ | Allow fallback recommendations (default: true) |

## Integration Examples

### JavaScript/Node.js

```javascript
async function getRecommendations(query) {
  const response = await fetch('https://api.useadmesh.com/agent/recommend', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.ADMESH_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      query: query,
      format: 'auto'
    })
  });

  const data = await response.json();
  return data.response.recommendations;
}

// Usage
const recommendations = await getRecommendations('best CRM for startups');
```

### Python

```python
import requests
import os

def get_recommendations(query: str):
    response = requests.post(
        'https://api.useadmesh.com/agent/recommend',
        headers={'Authorization': f'Bearer {os.environ.get("ADMESH_API_KEY")}'},
        json={
            'query': query,
            'format': 'auto'
        }
    )
    data = response.json()
    return data['response']['recommendations']

# Usage
recommendations = get_recommendations('best CRM for startups')
```

### React Component

```tsx
import React, { useState } from 'react';

function RecommendationSearch() {
  const [recommendations, setRecommendations] = useState([]);

  const searchRecommendations = async (query) => {
    const response = await fetch('/api/recommendations', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ query })
    });
    
    const recs = await response.json();
    setRecommendations(recs);
  };

  return (
    <div>
      <input 
        placeholder="Ask anything..."
        onKeyPress={(e) => {
          if (e.key === 'Enter') searchRecommendations(e.target.value);
        }}
      />
      
      {recommendations.map(rec => (
        <div key={rec.ad_id}>
          <h3>{rec.title}</h3>
          <p>{rec.meta.reason}</p>
          <a href={rec.admesh_link} target="_blank">Learn More</a>
        </div>
      ))}
    </div>
  );
}
```

## Displaying Recommendations

### Option 1: Use AdMesh UI SDK (Recommended)

Install the UI SDK for pre-built components:

```bash
npm install admesh-ui-sdk
```

```tsx
import { AdMeshLayout } from 'admesh-ui-sdk';

function MyApp() {
  const [recommendations, setRecommendations] = useState([]);

  return (
    <AdMeshLayout
      recommendations={recommendations}
      layout="auto"
      onRecommendationClick={(adId, admeshLink) => window.open(admeshLink)}
    />
  );
}
```

[Learn more about UI SDK →](/ui-sdk/installation)

### Option 2: Custom Implementation

Build your own UI using the recommendation data:

**Key Requirements**:
- ✅ Use `admesh_link` for all clicks (required for tracking)
- ✅ Display `title` and `reason` prominently
- ✅ Show `trust_score` or `contextual_relevance_score` for quality signals
- ✅ Include disclosure labels ("Ad", "Sponsored", "Partner Recommendation")

**Example Custom Card**:
```tsx
function RecommendationCard({ rec }) {
  return (
    <div className="recommendation-card">
      <span className="badge">Partner Recommendation</span>
      <h3>{rec.title}</h3>
      <p>{rec.meta.reason}</p>
      
      {rec.meta.pricing && <p className="pricing">{rec.meta.pricing}</p>}
      {rec.meta.has_free_tier && <span className="badge">Free Tier</span>}
      
      <a href={rec.admesh_link} target="_blank" className="cta-button">
        Learn More
      </a>
      
      <div className="metadata">
        <span>Trust Score: {rec.trust_score}%</span>
        <span>Relevance: {rec.meta.contextual_relevance_score}%</span>
      </div>
    </div>
  );
}
```

## Earnings Model

### How You Get Paid

You earn revenue when users engage with recommendations:

| Engagement | You Earn | Example |
|------------|----------|---------|
| **Exposure** | Share of CPX | ~$0.004 per exposure |
| **Click** | Share of CPC | ~$0.25 per click |
| **Conversion** | Share of CPA | ~$5.00 per conversion |

### Revenue Share

- **Default Split**: 50% to AI platform, 50% to AdMesh
- **Custom Splits**: Available for high-volume partners
- **Minimum Payout**: $50
- **Payment Schedule**: Monthly via Stripe

### Earnings Calculation

```
Your Earnings = (CPX + CPC + CPA) × Revenue Share %
```

**Example**:
- User sees recommendation (CPX: $0.008)
- User clicks (CPC: $0.50)
- User converts (CPA: $10.00)
- **Total**: $10.508 (only CPA is billed due to single billing)
- **Your Earnings**: $10.508 × 50% = **$5.25**

## Best Practices

### Query Handling

<CardGroup cols={2}>
  <Card title="Do's" icon="check">
    - Send natural language queries
    - Include conversation context
    - Use session IDs for tracking
    - Handle fallback gracefully
  </Card>
  
  <Card title="Don'ts" icon="x">
    - Don't send keyword-stuffed queries
    - Don't manipulate intent detection
    - Don't cache recommendations > 5 min
    - Don't modify tracking links
  </Card>
</CardGroup>

### Recommendation Display

- **Filter by CRS**: Only show recommendations with Contextual Relevance Score > 60
- **Rank by Score**: Display highest-scoring recommendations first
- **Limit Count**: Show 3-5 recommendations maximum
- **Clear Disclosure**: Label as "Ad", "Sponsored", or "Partner Recommendation"

### User Experience

- **Relevance First**: Only show recommendations when truly relevant
- **Non-intrusive**: Don't interrupt user flow
- **Clear CTAs**: Make it obvious what happens when user clicks
- **Fast Loading**: Recommendations should load in < 500ms

## Testing

### Test Mode

Use test mode to verify integration without affecting production metrics:

```json
{
  "query": "test query",
  "format": "auto",
  "is_test": true
}
```

Test recommendations will:
- ✅ Appear in dashboard with "Test" label
- ✅ Not count toward earnings
- ✅ Not deduct from brand budgets
